\section{Verified depth-first search}
\label{sec:dfs}

The pebble game of \cref{sec:pebble-game} repeatedly searches a
partial orientation for a vertex with a free pebble reachable from a
target endpoint, and then reverses the arcs of the witnessing path.
Its inner primitive is a depth-first search (DFS) on a directed graph
that stops at the first vertex satisfying a given predicate and returns
a simple path to it. Mathlib's notions of walk and reachability for
simple graphs ($\mathrm{Walk}$, $\mathrm{Reachable}$) are declarative
--- they assert that a walk or its reversal exists --- rather than an
algorithm that returns one, so we build the search here and use it as a
black box in \cref{def:tryReachPebble}.

This section proves the search sound and complete against the
reflexive-transitive closure of its arc relation
(\cref{thm:reachable-finding-correct}), then derives from it a
computable operator returning the entire set of vertices reachable from
a source (\cref{def:reachClosureComputable}). The pebble game's
soundness proof uses the search directly, and its completeness proof
uses the closure operator.

\begin{definition}[Directed walk]
  \label{def:directed-walk}
  \lean{CombinatorialRigidity.Search.DirectedWalk,
    CombinatorialRigidity.Search.DirectedWalk.IsPath}
  \leanok
  Let $V$ be a type and $R$ a relation on $V$. A \emph{directed
  $R$-walk} from $u$ to $w$ is a non-empty list $u = v_0, v_1, \ldots,
  v_m = w$ of vertices with $R\,v_i\,v_{i+1}$ for $0 \le i < m$. The
  walk is \emph{simple} if its vertices are distinct.
\end{definition}

\begin{definition}[Depth-first reachability search]
  \label{def:reachable-finding}
  \lean{CombinatorialRigidity.Search.reachableFinding}
  \leanok
  \uses{def:directed-walk}
  Let $V$ be finite with decidable equality, let $\mathrm{succ} : V \to
  \mathrm{List}\,V$ be an out-neighbour function presenting the relation
  $R\,a\,b = (b \in \mathrm{succ}\,a)$, and let $P$ be a Boolean
  predicate on $V$. Given a source vertex $v$, the search
  $\mathrm{reachableFinding}\,\mathrm{succ}\,P\,v$ performs a
  depth-first search along $R$-arcs from $v$, returning one of:
  \begin{itemize}
    \item $\mathrm{some}\,(w, p)$, at the first reachable vertex $w$
      with $P(w)$, where $p$ is a simple $R$-walk from $v$ to $w$; or
    \item $\mathrm{none}$, when no vertex in the $R$-reachability
      closure of $v$ satisfies $P$.
  \end{itemize}
  The search terminates because each recursive call marks one further
  vertex visited, so the number of not-yet-visited vertices strictly
  decreases.
\end{definition}

\begin{fmlnote}
  The out-neighbours are presented as a list rather than a finite set
  so that the search is computable: mathlib's conversion from a finite
  set to a list is noncomputable, and would propagate to the search
  body. The visited set is carried as a finite set only for the
  termination measure --- the count of not-yet-visited vertices --- and
  is never enumerated.
\end{fmlnote}

\begin{theorem}[Correctness of \texorpdfstring{$\mathrm{reachableFinding}$}{reachableFinding}]
  \label{thm:reachable-finding-correct}
  \lean{CombinatorialRigidity.Search.reachableFinding_sound,
    CombinatorialRigidity.Search.reachableFinding_complete}
  \leanok
  \uses{def:reachable-finding, def:directed-walk}
  Let $\mathrm{succ}$, $v$, $P$ be as in \cref{def:reachable-finding}.
  \begin{enumerate}
    \item (soundness) If $\mathrm{reachableFinding}\,\mathrm{succ}\,P\,v
      = \mathrm{some}\,(w, p)$, then $p$ is a simple $R$-walk from $v$
      to $w$ and $P(w)$ holds.
    \item (completeness) If some vertex in the $R$-reachability closure
      of $v$ satisfies $P$, then $\mathrm{reachableFinding}\,
      \mathrm{succ}\,P\,v = \mathrm{some}\,(w', p)$ for some such $w'$.
      The returned $w'$ need not be the vertex the hypothesis names,
      since the search returns the first match in its traversal order.
  \end{enumerate}
\end{theorem}
\begin{proof}
  \leanok
  \uses{def:reachable-finding}
  Both parts are by well-founded induction on the termination measure
  of \cref{def:reachable-finding}. For soundness, at each recursive
  step the walk extends by one arc, the predicate is tested at each
  newly visited vertex before recursion, and simplicity follows from
  the visited-set invariant: the search does not recurse into an
  already-visited vertex, so no vertex repeats.

  For completeness, suppose some reachable vertex $w$ satisfies $P$
  while the search returns $\mathrm{none}$. The reachability witness
  lifts to an explicit $R$-walk from $v$ to $w$ by induction on the
  reflexive-transitive-closure witness ($\mathrm{Relation.ReflTransGen}$),
  prepending one arc at the head; it remains to contradict the
  existence of such a walk.

  The core is the invariant that a search from visited set
  $\mathrm{visited}$ returning $\mathrm{none}$ admits no $R$-walk
  avoiding $\mathrm{visited}$ that ends at a $P$-satisfying vertex; at
  $\mathrm{visited} = \varnothing$ this contradicts the walk just
  built. The invariant is proved by induction on the search's
  recursion, case-splitting on whether the current vertex is visited,
  satisfies $P$, or neither, with an inner induction on a bound for the
  walk length. When the walk revisits the current source vertex,
  truncating it at the first such occurrence
  ($\mathtt{DirectedWalk.dropUntilBundle}$, which returns the shorter
  sub-walk together with its length and vertex-set bounds) gives a
  strictly shorter walk closed by the inner induction hypothesis;
  otherwise the walk's tail avoids the enlarged visited set and the
  outer induction hypothesis closes the case.
\end{proof}

The pebble game's \cref{def:tryReachPebble} specialises this search:
it takes $R$ to be the outgoing-arc relation of the current partial
orientation and $P$ to be ``carries a free pebble'', and on success
reverses every arc of the returned walk. The search itself makes no
reference to the pebble game and is reusable wherever reachability with
a witness path is needed; the arc-reversal step is specific to the
pebble game.

\subsection{Reachability closure}
\label{ssec:reach-closure-computable}

The pebble game's completeness proof needs a different view of
reachability from $v$: not the first vertex matching a predicate, but
the whole set of vertices reachable from $v$. We record this as a
computable operator $\mathrm{reachClosureComputable}$ returning a finite
set of vertices, characterised
(\cref{lem:mem-reachClosureComputable}) as the reflexive-transitive
closure of the $\mathrm{succ}$-arc relation. This membership
characterisation is the contract the pebble game relies on in
\cref{sec:pebble-game}.

\begin{definition}[Reachability closure]
  \label{def:reachClosureComputable}
  \lean{CombinatorialRigidity.Search.reachClosureComputable}
  \leanok
  \uses{def:reachable-finding}
  Let $V$ be finite with decidable equality and $\mathrm{succ} : V \to
  \mathrm{List}\,V$ an out-neighbour function presenting the relation
  $R\,a\,b = (b \in \mathrm{succ}\,a)$. Then
  $\mathrm{reachClosureComputable}\,\mathrm{succ}\,v$ is the finite set
  of vertices $w$ for which $\mathrm{reachableFinding}\,\mathrm{succ}\,
  (\cdot = w)\,v$ succeeds; that is, the vertices reachable from $v$
  along $R$. The membership test is the decidable success of the
  search, so the operator is computable.
\end{definition}

\begin{lemma}[Membership in the reachability closure]
  \label{lem:mem-reachClosureComputable}
  \lean{CombinatorialRigidity.Search.mem_reachClosureComputable,
    CombinatorialRigidity.Search.reachClosureComputable_sound,
    CombinatorialRigidity.Search.reachClosureComputable_complete}
  \leanok
  \uses{def:reachClosureComputable, thm:reachable-finding-correct}
  For any $\mathrm{succ}$ and vertices $v, w$,
  \[
    w \in \mathrm{reachClosureComputable}\,\mathrm{succ}\,v
      \;\Longleftrightarrow\;
      \mathrm{Relation.ReflTransGen}\,
        (\lambda a\,b \mapsto b \in \mathrm{succ}\,a)\,v\,w.
  \]
\end{lemma}
\begin{proof}
  \leanok
  \uses{def:reachClosureComputable, thm:reachable-finding-correct}
  For the forward direction, a successful search yields a witness walk
  (\cref{thm:reachable-finding-correct}), which lifts to the
  reflexive-transitive closure by a one-step induction
  ($\mathtt{DirectedWalk.toReflTransGen}$). For the reverse direction,
  given the reachability chain, run the search at the predicate
  $(\cdot = w)$ with $w$ as the existence witness;
  \cref{thm:reachable-finding-correct} then returns $\mathrm{some}$, so
  $w$ meets the membership condition of
  \cref{def:reachClosureComputable}.
\end{proof}
